关于
糖包是什么

糖包是一款本地运行的 AI 助手桌面端（基于 Electron）。它把多种 AI 能力整合到左侧导航的「模块」里，数据保存在你本机，不强制上传云端。

模块一览
💬
糖包 · 聊天
多轮对话。顶部可切换模型、开启「深度思考」与「联网搜索」，对话可通过顶部 ⋯ 菜单导出或复制为 Markdown。
🎨
糖绘 · 图像
输入文字描述生成图片。
📖
糖读 · 文档
上传文档后，可对内容进行总结、提取要点、翻译、生成提纲等分析。
🤖
糖创 · 智能体
设计与管理智能体（Agent）。内置写作、翻译、代码等预设，可自建智能体并设置系统提示词、语气与模型；另含模板库与工作流。
⚡
糖码 · 本地智能体
本地运行的 agentic 助手（类似 Claude Code）。在限定工作目录内自主调用工具——读写文件、执行命令、运行 git、联网检索；以「项目 + 多会话」组织，每步受权限管控。
快速上手
  1. 打开左下角齿轮 →「API 与密钥 / 账户」，新增一个账户，填写 API Base URL、API Key 与模型。
  2. 在「API 与密钥」中选择该账户作为默认（或按需为各模块单独指定）。
  3. 从左侧导航切换模块，直接开始对话或创作。
  4. 在「模块」面板里可开关内置模块、或添加自定义网页模块。
  5. 使用「糖码」时，先新建项目并配置工作目录与权限，再开始编码。
配置
API 与密钥

在「API 与密钥」先选模块（默认 / 糖包 / 糖绘 / 糖读），再到「账户」新增账户：API Base URL、API Key、模型列表。可设一个默认账户，也允许各模块独立账户。

⚠️ 密钥仅保存在本机浏览器 localStorage，不会上传到糖包服务器；但为明文存储，请勿在公共电脑导出含 Key 的配置给他人。

自定义模块

在「模块」面板底部添加自定义模块：填写名称与网页 URL（http/https），它会以 iframe 嵌入并出现在左侧导航。切换模块不会重载网页；若页面需要刷新，可点顶栏的刷新按钮手动刷新。

外观与数据

「外观」可切换浅色 / 深色 / 跟随系统，自定义强调色与界面圆角，修改即时生效并在刷新后保持。

「数据」可导出 / 导入配置（账户、智能体、模板、工作流、提示词、外观、模块开关）。导入会覆盖现有设置。

权限
糖码项目权限

糖码以「项目」为单位管理权限，每个项目可设置：

  • 自动执行（auto）：开启后工具调用无需逐个确认；关闭则需你每次审批。
  • 需审批工具（approveTools）：列出必须人工确认后才执行的工具。
  • 命令白名单（cmdWhitelist）：仅允许执行白名单内的命令，未列出的命令将被拦截。

所有工具调用都以本地安全围栏（safePath）为硬约束，越权操作会被拒绝。

常见问题
  • 请求被 CORS 拦截？浏览器直连第三方 API 可能受跨域限制。请将 Base URL 指向支持跨域的端点，或自建本地代理。
  • 怎么换模型？ 聊天模块顶部点模型按钮切换；也可在「API 与密钥」里为模块指定默认模型。
  • 导出配置为什么提示含 Key？ 导出文件包含 API Key 明文，请勿分享给他人。
  • 自定义模块打不开？ 请确认 URL 为 http/https 且目标站点允许被 iframe 嵌入（部分站点设置了 X-Frame-Options 会拒绝）。
隐私提示

⚠️ 你的对话、文档、密钥均保存在本机 localStorage，糖包不会主动上传。请自行负责本地数据安全与备份。
